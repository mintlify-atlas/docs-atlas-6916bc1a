## Using Browser Tools

Most tools in this collection are HTML/JavaScript applications that run entirely in your browser. They require no installation and process everything locally for privacy.

### Finding Browser Tools

Browser tools are organized by category on the main site:

- **Image & Media** - Image cropping, OCR, SVG tools, video processing
- **Text & Documents** - Markdown rendering, PDF tools, clipboard utilities  
- **Data & Time** - Timestamp conversion, timezone comparison, date calculators
- **LLM Playgrounds** - Interact with various AI models
- **Bluesky & Social** - Social media tools and firehose viewers
- **GitHub & Dev** - Development utilities and API tools

### Opening a Browser Tool

<Steps>
  <Step title="Navigate to the tool">
    Visit any tool at `https://tools.simonwillison.net/[tool-name]`
    
    For example: [https://tools.simonwillison.net/ocr](https://tools.simonwillison.net/ocr)
  </Step>
  
  <Step title="Use the tool interface">
    Most tools provide a simple interface:
    - Drag and drop files into the upload area
    - Paste content directly from your clipboard
    - Click to select files from your device
  </Step>
  
  <Step title="Process locally">
    Files are processed entirely in your browser using JavaScript. Nothing is uploaded to a server.
  </Step>
  
  <Step title="Download or copy results">
    Results can typically be:
    - Downloaded as files
    - Copied to your clipboard
    - Viewed directly in the browser
  </Step>
</Steps>

### Example: Using the OCR Tool

The OCR tool demonstrates typical browser tool usage:

<CodeGroup>

```text Basic Usage
1. Visit https://tools.simonwillison.net/ocr
2. Drag and drop a PDF or image file
```

</Tip>

## Running Python Scripts

Python scripts can be run directly from their URLs using [uv](https://docs.astral.sh/uv/), Astral's Python package installer and resolver. No git clone or local installation required.

### Prerequisites

<Steps>
  <Step title="Install uv">
    Install uv using one of these methods:
    
    <CodeGroup>
    
    ```bash macOS/Linux
    curl -LsSf https://astral.sh/uv/install.sh | sh
    ```
    
    ```bash Windows
    powershell -c "irm https://astral.sh/uv/install.ps1 | iex"
    ```
    
    ```bash Homebrew
    brew install uv
    ```
    
    ```bash pip
    pip install uv
    ```
    
    </CodeGroup>
  </Step>
  
  <Step title="Verify installation">
    ```bash
    uv --version
    ```
    You should see output like `uv 0.5.0` or similar.
  </Step>
</Steps>

### Running a Python Script

<Steps>
  <Step title="Find a script URL">
    All Python scripts are available at:
    ```
    https://tools.simonwillison.net/python/[script-name].py
    ```
  </Step>
  
  <Step title="Run with uv run">
    Execute the script directly from its URL:
    ```bash
    uv run https://tools.simonwillison.net/python/[script-name].py [arguments]
    ```
  </Step>
  
  <Step title="uv handles dependencies">
    uv automatically:
    - Downloads the script
    - Reads inline dependency metadata
    - Creates an isolated environment
    - Installs required packages
    - Runs the script
  </Step>
</Steps>

### Example: JSON Extractor

Extract valid JSON objects from mixed text:

<CodeGroup>

```bash From File
uv run https://tools.simonwillison.net/python/json_extractor.py README.md
```

```bash From stdin
curl 'https://example.com/page-with-json' | \
  uv run https://tools.simonwillison.net/python/json_extractor.py
```

```bash All JSON Objects
uv run https://tools.simonwillison.net/python/json_extractor.py \
  --all README.md
```

```bash Get Help
uv run https://tools.simonwillison.net/python/json_extractor.py --help
```

</CodeGroup>

### Example: Claude Transcript Converter

Convert Claude JSONL conversation logs to Markdown:

<CodeGroup>

```bash Basic Conversion
uv run https://tools.simonwillison.net/python/claude_to_markdown.py \
  conversation.jsonl
```

```bash Custom Output
uv run https://tools.simonwillison.net/python/claude_to_markdown.py \
  conversation.jsonl output.md
```

</CodeGroup>

<Info>
The script automatically creates a `.md` file next to the `.jsonl` file if no output path is specified.
</Info>

### Example: HTTP Checker

Check if a URL supports gzip, ETags, and conditional GET:

```bash
uv run https://tools.simonwillison.net/python/http_check.py \
  https://example.com/api/data.json
```

### Scripts with API Keys

Some scripts require API keys. Set them as environment variables:

<CodeGroup>

```bash OpenAI
OPENAI_API_KEY=$(llm keys get openai) \
  uv run https://tools.simonwillison.net/python/openai_image.py \
  'A raccoon wearing a hat'
```

```bash Mistral OCR
MISTRAL_API_KEY=your-key-here \
  uv run https://tools.simonwillison.net/python/mistral_ocr.py \
  document.pdf > output.md
```

</CodeGroup>

## Common Patterns

### Browser Tools

<AccordionGroup>
  <Accordion title="Privacy: Where is my data processed?">
    Most browser tools use client-side JavaScript libraries to process everything locally in your browser. Your files never leave your device. Tools that make API calls (like LLM playgrounds) clearly indicate this in their interface.
  </Accordion>
  
  <Accordion title="Source Code: How do I see how it works?">
    Every browser tool includes a link to its source code on GitHub, usually at the bottom of the page. The code is typically a single HTML file containing all JavaScript inline.
  </Accordion>
  
  <Accordion title="Offline Usage: Can I save and use tools offline?">
    Yes! Use your browser's "Save Page As" feature to save the complete HTML file. Most tools will work offline, except those requiring external API calls.
  </Accordion>
</AccordionGroup>

### Python Scripts

<AccordionGroup>
  <Accordion title="Dependencies: How are packages installed?">
    Scripts use PEP 723 inline dependency metadata. When you run `uv run`, it reads the `# /// script` block in the file, creates an isolated environment, and installs dependencies automatically.
  </Accordion>
  
  <Accordion title="Caching: Are scripts cached?">
    Yes, uv caches both the scripts and their dependencies. Subsequent runs are much faster.
  </Accordion>
  
  <Accordion title="Help Text: How do I see all options?">
    Most scripts support `--help`:
    ```bash
    uv run https://tools.simonwillison.net/python/[script].py --help
    ```
  </Accordion>
  
  <Accordion title="Local Development: Can I modify scripts?">
    Absolutely! Download the script, modify it, and run it locally:
    ```bash
    curl -O https://tools.simonwillison.net/python/script.py
    # Edit script.py
    uv run script.py [arguments]
    ```
  </Accordion>
</AccordionGroup>
